1. 자바프로그램 시작 -> 자바 파일 생성 (클래스)

2. 코드가 많아지면 함수를 이용해서 쪼갠다.
   변수가 많아지면 클래스를 이용해서 쪼갠다.

3. 클래스에 비슷한 성격의 데이터와 함수를 묶는다.

4. main에서 불러서 사용
  -> class를 직접 사용하는 방법 (static)
  -> class의 복사본인 객체를 생성해서 사용하는 방법 (non - static)

5. 객체의 개성 (나이, 이름, 거주지) 을 표현할 때는 non - static
   모든 객체가 동일하게 사용하는 공유 정보 (인류 평균 나이) 는 static
   -> static이 붙은 것은 static이 궅지 않은 것을 사용할 수 없다. 반대는 가능하다.

데이터의 중복 -> 변수
코드의 중복 -> 함수
함수의 중복 -> 클래스
클래스의 중복 -> 클래스, 객체
객체의 중복 -> 상속

상속
  - class 자식 extends 부모
ex)
class Animal { 중복객체 }
class Dog extends Animal { }
class Cat extends Animal { }

메서드 오버라이드 ( 재정의 ) => 부모한테 물려받은 메서드와 동일한 이름의 메서드를 자식이 다시 정의하는 것
자신에게 가까운 메서드를 우선적으로 사용.
